SMTP, POP e IMAP: Cómo funciona el correo electrónico

SMTP (Protocolo Simple de Transferencia de Correo) se utiliza para la gestión del envío de correos.

Para soportar los servicios de correo electrónico, se requieren al menos dos protocolos: SMTP y POP/IMAP. Juntos permiten el envío y recepción de correos.

Funciones básicas del servidor SMTP:

  • Verifica quién está enviando los correos a través del servidor SMTP.
  • Envía los correos salientes.
  • Si el correo no se puede enviar, retorna un mensaje de error al remitente.


POP e IMAP

Ambos permiten que programas como Outlook, Thunderbird o apps de correo móvil se comuniquen con el servidor donde llegan los mensajes.


📮 POP (Post Office Protocol)

  • Descarga los correos del servidor al dispositivo y luego los elimina (salvo que se configure lo contrario).
  • Los correos se guardan solo en el dispositivo donde se descargaron.
  • No se sincroniza con otros dispositivos.
  • Permite leer sin conexión, pero los cambios no se reflejan en otros dispositivos.

Ejemplo: POP es como sacar todas las cartas del buzón y llevárselas. El buzón (servidor) queda vacío, y otros dispositivos no verán nada.


🗂 IMAP (Internet Message Access Protocol)

  • Los correos se mantienen en el servidor; solo se descarga una copia temporal para mostrar en el dispositivo.
  • Permite sincronización total entre todos los dispositivos.
  • Ideal para acceder al correo desde múltiples lugares.
  • Requiere conexión a Internet para mayor funcionalidad.

Ejemplo: IMAP es como abrir el buzón, leer las cartas, marcarlas o moverlas directamente ahí. Si accedes desde otro dispositivo, todo estará igual.



⚙️ Proceso de envío con SMTP

  1. El agente de usuario (cliente o programa externo) se conecta al servidor SMTP (por ejemplo, smtp.google.com) a través del puerto 25.
  2. Se inicia la sesión SMTP tras validar la conexión.
  3. El cliente envía las direcciones del remitente y destinatario, el cuerpo del mensaje y adjuntos.
  4. El servidor SMTP verifica si los dominios coinciden.
  5. Si no, el servidor SMTP del remitente se conecta al del destinatario. Si no hay conexión, el mensaje se coloca en una cola SMTP.
  6. El servidor del destinatario verifica dominio y nombre de usuario, y reenvía al servidor POP o IMAP.
  7. Finalmente, el correo aparece en la bandeja de entrada del destinatario.

Este proceso es una versión simplificada. Puedes leer más en: computer.howstuffworks.com (correo electrónico)

¿Se sincroniza con el servidor? No Sí
¿Depende de conexión a internet? No siempre Casi siempre
