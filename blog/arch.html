Instalación y configuración de Arch Linux (con entorno GNOME y VMware)


Es hora de configurar tu propio entorno en Arch Linux

Instalación y configuración de máquina virtual


• Instalar la ISO de Arch

Primero vamos a instalar la ISO de Arch. Existen varias formas de hacerlo; yo por ejemplo prefiero descargarla directamente desde uno de los mirrors oficiales para evitar el uso de torrents:

https://archlinux.org/download/#http-downloads

Al ingresar a ese link, yo escogí el primer mirror de España y descargué la ISO más reciente: archlinux-2025.10.01-x86_64.iso.

A mí me gusta mantener los datos ordenados, así que creo una carpeta llamada Imágenes donde guardo todos los ISOs que descargo. Esto me permite tener control sobre las versiones y evitar eliminaciones accidentales.

Este hábito me ha ayudado a no borrar por accidente imágenes y tener que volver a descargarlas o perder tiempo buscando versiones.

Una vez creada la carpeta y movida la ISO dentro de ella, prefiero crear otra carpeta separada que contendrá todos los datos de la Máquina Virtual.

En este ejemplo he creado una carpeta llamada Prueba para seguir el flujo de creación de la máquina virtual más fácil.


• Creación de la Máquina Virtual (VMware)

En este caso estoy usando VMware porque me ha resultado estable. Dentro del virtualizador hay dos formas de crear una VM:

  1. Dando click en File → New Virtual Machine.
  2. Usar el botón principal de “Crear nueva máquina virtual”.

Se abrirá una ventana para configurar recursos. Yo suelo seleccionar la opción simple, pero son libres de usar la segunda opción si saben como usarla.

En la pantalla siguiente elegimos la imagen ISO con Browse… y buscamos la ISO dentro de la carpeta donde la guardamos:

Una vez elegida la ISO, confirmamos la selección y continuamos.

En la parte donde se nos pide el sistema operativo a virtualizar, Arch no suele aparecer en la lista; seleccionar Linux → Ubuntu funciona sin problema.


Nombre y ubicación de la VM

Nombre: simplemente es cómo aparece en la biblioteca. No influye en el sistema instalado.

Ubicación: aquí se guardará el disco virtual (vmdk/vdi). Yo recomiendo usar un disco externo si no quieres llenar tu disco principal.


Tamaño del disco y almacenamiento

El tamaño depende del uso:

  • Laboratorio rápido: 20 GB.
  • Uso prolongado (descargas, herramientas): 90 GB.

Recomiendo usar un solo archivo para el disco virtual por rendimiento, salvo que necesites fragmentarlo por compatibilidad.

En la ventana final, podemos ver un breve resumen de la configuración final de nuestra Máquina Virtual. Yo suelo finalizar el asistente y luego editar los ajustes antes de arrancar la VM.

Ajustes previos al inicio

Antes de iniciar la máquina, recomiendo:

  • Desactivar la aceleración gráfica (opcional); puede causar problemas con la terminal o herramientas como OBS.
  • Ajustar la RAM sin exceder el 50–60% del total del equipo físico.
  • Asignar procesadores de forma responsable (ejemplo: 4 procesadores x 2 cores = 8 vCPUs si tu host lo permite).

Resultado final:

Una vez configurado todo, iniciamos la VM para comenzar la instalación.


Configuración de discos

Cuando la ISO arranca, verás una consola. Lo primero es configurar el teclado y comprobar la red.


Configurar teclado

Listamos los keymaps y cargamos el apropiado (ejemplo: español latino):

$ localectl list-keymaps
$ loadkeys la-latin1

Comprobar conexión de red

Hacemos un ping para confirmar conectividad con el exterior:

$ ping -c 3 archlinux.org

Particionado con cfdisk

Usaremos cfdisk y crearemos particiones según el uso. Si la interfaz de cfdisk va lenta en la VM, desactiva la aceleración gráfica y vuelve a intentarlo.

$ cfdisk

Ahora escogeremos la opción dos:


Ejemplo de esquema (para un disco ~90 GB):

  • /dev/sda1 — 512M — vfat — /boot
  • /dev/sda2 — 85G — ext4 — /
  • /dev/sda3 — 4.5G — swap

Una vez seleccionado le damos a la opción de New para crear una nueva partición:


Primero iniciamos con 512M le damos Enter para seleccionar que sea una partición primaria


De vuelta a Free space, vamos a crear una nueva partición:


Y para el total de espacio que quede le vamos a restar 4.5 por ejemplo de 19.5 se agregaría un total de 15

En este caso mi espacio total es de 89.5 entonces al restar el 4.5 queda un total de 85. Ese va a ser el total del nuevo disco


Luego lo seleccionamos como partición primaria


Ahora con el espacio restante que serían los 4.5G los vamos a usar como partición primaria la cual va a corresponder al swap


Selección de tipos de particiones


A la partición de 4.5 le vamos a asignar el tipo de swap


Ahora escogemos la opción de swap solaris

Luego de haber seleccionado eso procedemos a darle Write y luego Quit

Al darle write le deben de ingresar yes para que los cambios se ejecuten:


Ahora podemos confirmar los cambios del disco con el comando lsblk


Formateo y tipos

Vamos a hacer uso de mkfs.vfat para indicar de que queremos formatear una partición en formato vfat:

  • Hacemos uso de mkfs.vfat -F 32 /dev/sda1 para hacer uso de la partición primaria de 512M
  • Ahora vamos a hacer lo mismo con el sda2 mkfs.ext4 /dev/sda2 la cual es la partición más grande
$ mkfs.vfat -F 32 /dev/sda1
$ mkfs.ext4 /dev/sda2

Con mkswap vamos a indicarle la partición correspondiente la cual va a asignar el swap

  • Recordar que el swap lo dejamos en el sda3
  • Luego hacemos swapon para aplicar los cambios
$ mkswap /dev/sda3
$ swapon

Montaje

Empezamos con el proceso de montado

  • El primer paso va a ser agregar todo el sda2 a mnt con mount (La partición grande)
  • Luego vamos a crear un directorio llamado boot dentro de mnt
  • Ahora lo que teníamos en sda1 lo vamos a montar en boot
$ mount /dev/sda2 /mnt
$ mkdir -p /mnt/boot
$ mount /dev/sda1 /mnt/boot

Instalación base

Instalamos paquetes base y herramientas esenciales:

$ pacstrap /mnt linux linux-firmware networkmanager grub wpa_supplicant base base-devel

Fstab

Generamos el archivo fstab:

$ genfstab -U /mnt > /mnt/etc/fstab

Definición de usuarios de sistema

Para eso usamos el comando arch-chroot /mnt para ingresar ya a lo que sería el sistema y así poder configurar usuarios, locales y bootloader:

$ arch-chroot /mnt

Usuarios y sudo

Crear usuario, asignar contraseña y darle permisos sudo:

  • Vamos a usar el comando passwd para establecer una nueva contraseña para el usuario root
  • Luego creamos el usuario que vamos a crear como usuario local par aluego asignarle una contraseña
  • Como en Arch todo funciona distinto y no tenemos sudo, necesitamos unir el usuario al grupo wheel para permisos sudo
  • También podemos proceder e instalar el paquete sudo y un editor para poder cambiar las configuraciones de los archivos
# Establecer contraseña root
$ passwd

# Crear usuario y añadir a wheel
$ useradd -m kallrize
$ passwd kallrize

# Instalar editores
$ pacman -S vim nano

Al final podemos comprobar que tenemos un home para nuestro usuario


Como en Arch todo funciona distinto y no tenemos sudo, necesitamos unir el usuario al grupo wheel para permisos obtener permisos como sudo

usermod -aG wheel kallrize

También podemos proceder e instalar el paquete sudo

pacman -S sudo

Ahora procedemos a abrir el archivo /etc/sudoers para editar los permisos

nano /etc/sudoers

Una vez dentro debemos de encontrar y descomentar esta linea para que siempre pida contraseña al hacer sudo y que se vea de la siguiente forma


Localización y teclado persistente

Siguiendo como root vamos a tocar las regiones en el /etc/locale.gen:

$ nano /etc/locale.gen   # descomentar es_ES.UTF-8 o cualquiera que se encuentre cerca de su región

Una vez eso se ha aplicado hacemos un locale-gen

$ locale-gen

Para no estar configurando el teclado cada vez que iniciamos vamos a hacerlo persistente de esta forma:

# Dentro se coloca KEYMAP=la-latin1
  nano /etc/vconsole.conf


Con esto cada vez que se inicie el teclado debería de estar listo con la configuración correcta


configuración del boot loader (Se recomienda hacer un snapshot en este punto)

Instalamos y configuramos GRUB:

$ grub-install /dev/sda
$ grub-mkconfig -o /boot/grub/grub.cfg

Hostname y hosts

Si hacemos un cat /etc/hostname se puede ver que el archivo no se encuentra por lo que hay que crearlo para que el pc se llame igual que el usuario o con cualquier otro nombre de elección

$ echo atlas > /etc/hostname
$ nano /etc/hosts
# agregar:
127.0.0.1    localhost
127.0.0.1    atlas.localdomain atlas

Una vez hemos terminado estos pasos, vamos a hacer un reboot del sistema para comprobar que todo se haya ejecutado de manera correcta:

$ exit
$ reboot now

Configuración de red y entorno gráfico (Se recomienda otro snapshot aquí)


Activación de red

Si no hay conectividad tras el primer arranque, habilitamos NetworkManager y wpa_supplicant:

$ sudo systemctl start NetworkManager.service
$ sudo systemctl enable NetworkManager.service
$ sudo systemctl start wpa_supplicant.service
$ sudo systemctl enable wpa_supplicant.service

Acceso a AUR y repositorios BlackArch

Ahora el siguiente paso sería aumentar el alcance de paquetes disponibles en el sistema

Para esto la idea es poder acceder a AUR el cual es un sistema de paquetes soportado por la comunidad de Arch

Instalamos herramientas para compilar AUR helpers y añadir BlackArch como usuario local por lo que salimos de root usando exit:

$ sudo pacman -S git xdg-user-dirs
$ sudo -u kallrize xdg-user-dirs-update

Posicionandonos en el directorio personal de usuario vamos a crear un directorio nuevo dentro de desktop con nuestro nombre y otro llamado repos donde vamos aguardar los repositorios de paquetes

Luego ingresamaos dentro del directorio repos y procedemos a descargar el siguiente repositorio usando git clone:

  • https://aur.archlinux.org/paru-bin.git
# En el directorio del usuario
$ mkdir -p ~/Desktop/kallrize/repos
$ cd ~/Desktop/kallrize/repos
$ git clone https://aur.archlinux.org/paru-bin.git
$ cd paru-bin
$ makepkg -si

Para agregar BlackArch:

Una vez tenemos el AUR listo procedemos con la descarga de los repositorios de blackArch para la descarga de las herramientas de pentest (este paso es totalmente opcional)

El directorio blackArch se debe de crear dentro del directorio repos

$ mkdir blackarch
$ cd blackarch
$ curl -O https://blackarch.org/strap.sh
$ chmod +x ./strap.sh
$ sudo ./strap.sh
$ sudo pacman -Sy

Instalación del entorno gráfico (GNOME)

Instalamos Xorg y GNOME, y habilitamos GDM:

$ sudo pacman -S xorg xorg-server # Todo Enter
$ sudo pacman -S gnome # Todo Enter
$ sudo systemctl start gdm.service

Como se puede ver, las proporciones no son correctas, por lo que más adelante vamos a reparar esta área

Otro problema es que la terminal no es funcional por lo que debemos de regresar al estado de terminal de antes

Para esto vamos a usar las teclas ctrl + alt + f3 y salimos del modo de interfaz gráfica para operar nuevamente en la terminal

Una vez dentro de la terminal, vamos a aprovechar para dejar habilitado el servicio de gdm

$ sudo systemctl enable gdm.service

Aprovechando que estamos ahi, vamos a descargar una terminal funcional

Para una terminal más estable recomiendo instalar Kitty:

$ sudo pacman -S kitty

Corrección de proporciones y VMware Tools

Para ajustar el redimensionamiento de pantalla e integración con VMware:

$ sudo pacman -S gtkmm open-vm-tools xf86-video-vmware xf86-input-vmmouse #Si alguno da errores de que el paquete no existe pero si se escribió mal, se puede ignorar y  seguir
$ sudo systemctl enable vmtoolsd
$ sudo reboot now

Tras reiniciar verifica que la resolución sea correcta. Como la terminal nativa de GNOME presenta problemas, usa kitty o la que hayas configurado.

Es probable que nuevamente el teclado se haya desconfigurado, pero esta vez lo podemos configurar fácilmente desde GUI en el apartado de configuración

Notas finales, trucos y conclusión

- Evita asignar más del 50–60% de la RAM del host a la VM. - Para entornos livianos 2–4 GB está bien; para GNOME recomiendo 4–8 GB. - Para CPUs, asigna recursos sin dejar al host sin margen. - Usa discos en un solo archivo para mejor rendimiento, salvo que necesites portabilidad.

- Si experimentas lentitud en cfdisk o entradas lentas, desactiva la aceleración gráfica de la VM antes de usar la consola. - Para terminal, instalo kitty por estabilidad y características.

Con esto tendrás una instalación funcional de Arch Linux en VMware, lista para personalizar (temas, utilidades, herramientas pentest desde BlackArch/AUR).
