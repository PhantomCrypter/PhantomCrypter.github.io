Personalización de entorno Arch Linux

Es hora de personalizar tu propio entorno en Arch Linux

Configuración de entorno bspwm


Instalación de paquetes base

Primero vamos instalar los paquetes base para que nuestro entorno funcione correctamente: 

$ sudo pacman -S bspwm sxhkd

Estos paquetes son esenciales para el funcionamiento de bspwm y la gestión de atajos de teclado.


  • bspwm → el gestor de ventanas
  • sxhkd → demonio de atajos de teclado (indispensable)

Cuando ya tengamos todos los archivos listos procedemos con la configuración de todos los archivos. Para esto vamos a:

  • Crear el directorio bspwm y sxhkd
  • Copiar los archivos de configuración de ejemplo dentro de los dorectorios creados
  • Finalizamos asignándole permisos de ejecución al archivo bspwmrc
$ mkdir -p ~/.config/bspwm
$ mkdir -p ~/.config/sxhkd
$ cp /usr/share/doc/bspwm/examples/bspwmrc ~/.config/bspwm/
$ cp /usr/share/doc/bspwm/examples/sxhkdrc ~/.config/sxhkdrc
$ chmod +x ~/.config/bspwm/bspwmrc


Ahora tenemos que hacer una rápida configuración en el sxhkdrc para que cuando ingresemos a nuestro nuevo entorno se nos pueda abrir una terminal

$ nano ~/.config/sxhkd/sxhkdrc

Dentro del archivo vamos a buscar la siguiente línea y vamos a colocar la terminal que nosotros vamos a utilizar, en mi caso kitty:

Una vez instalados esos archivos vamos a proceder con la descarga de las mejoras de entorno gráfico

Estas herramientas nos van a permitir mejorar la estéetica del entorno

$ sudo pacman -S rofi polybar picom feh

En caso de que el Firefox no se encuentre instalado, lo podemos descargar de esta forma:

$ sudo pacman -S firefox

Luego vamos a salir del login para dar click en logo de configuración para seleccionar la opción de bspwm y asi poder ingresar con el nuevo entorno


Configuración de entorno

En caso de volver a tener problemas con el teclado lo podemos volver a solucionar de esta forma

$ setxkbmap latam #En mi caso es latam pero tambien puede ser "es"

Una vez tengamos el teclado configurrado vamos a proceder a ingresar al sxhkdrc para así poder agregar un shortcut a firefox y además hacer configuraciones de gestión de ventanas

$ nano ~/.config/sxhkd/sxhkdrc

Cuando estemos dentro vamos a buscar las siguientes configuraciones las cuales van a estar conguradas con letras (h,j,k,l) y no con flechas (Left,Down,Up,Right) y al estar virtualizando podría generar conflicto

Por ejemplo una de las configuraciones provocaba que bloqueara la pantalla de mi windows...

Configurrración de picom

Para iniciar con la configuración de picom, primero vamos a proceder a crear sus directorios de configuración para luego copiar los archivos de ejemplo dentro de la nueva carpeta picom

$ mkdir -p ~/.config/picom
$ cp /usr/share/doc/picom/examples/picom.conf ~/.config/picom/

En este apartado vamos a confirmar que se esté haciendo uso de xreder para evitar errores gráficos.

En caso de que se esté usando un entorno físico, es decir, un entorno no virtualizado, se puede hacer uso de glx sin ningún problema


Ahora, vamos arrancar picom de la siguiente forma: 

$ picom --backend xrender --no-vsync --no-use-damage &

el script depende de las configuraciones que escogimos en el archivo de configuración por ejemplo el backend y vsync


Configuración de Kitty:

Para poder iniciar con la configuración de la terminal, debemos de ingrresar al directorio /.config/kitty/

y ahí dentro crear un archivo llamado kitty.conf

Dentro de ese archvio vamos a colocar las siguientes líneas:

font_size 12.0
cursor_shape beam
window_padding_width 8
background_opacity 0.9

Además de esto a mi me gusta agregar las siguientes líneas

enable_audio_bell no
include color.ini #Si da problemas al agregar temas se elimina
font_family Cartograph CF Italic
font_size 11
disable_ligatures never
url_color #fff
url_style curly
shell zsh

Con esto ya tendremos una configuración básica de kitty, pero si queremos agregarle más personalización podemos descargar temas de la siguiente forma:

Instalación y configuración de la zsh:

La instalción de la zsh es muy simple, para eso solo debemos de usar el siguiente comando

$ sudo pacman -S zsh

Una vez instalada procedemos a cambiar la shell por defecto a zsh como usuario root:

# Se debe de ejecutar como root
$ usermod --shell /bin/zsh kallrize

Ahora, si cerramos la terminal y la abrimos nuevamente deberíamos de ver el cambio a zsh con el comando echo $SHELL


Descarga de Plugins:

En zsh podemos descargar una gran cantidad de plugins para poder mejorar ya sea la estética o funcionalidad de la Shell, en este caso a mí me gusta usar estas 2:

    zsh-syntax-hightlighting zsh-autosuggestions 
$ paru -S zsh-syntax-hightlighting zsh-autosuggestions

Para lo siguiente, vamos a necesitar instalar locate para poder ubicar los 2 plugins que acabamos de instalar

$ sudo pacman -S mlocate

Ahora procedemos a actualizar la base de datos de locate

$ sudo updatedb

Ahora si procedemos a buscar las rutas de los plugins

$ locate zsh-syntax-highlighting
$ locate zsh-autosuggestions

Con las rutas ya ubicadas procedemos a abrir el archivo .zshrc para agregar los plugins

$ nano ~/.zshrc

Dentro del archivo vamos a agregar las siguientes líneas al final del documento:

source /usr/share/zsh/plugins/zsh-syntax-highlighting/zsh-syntax-highlighting.zsh
source /usr/share/zsh/plugins/zsh-autosuggestions/zsh-autosuggestions.zsh

Instalación y configuración de lsd y bat:

Para instalar lsd y bat usamos el siguiente comando:

$ pacman -S lsd bat

Ahora procedemos a abrir el archivo .zshrc para agregar los alias de lsd y bat

$ nano ~/.zshrc

Dentro del archivo vamos a agregar las siguientes líneas al final del documento:

alias ls='lsd'

Yo el bat no me gusta agregarlo como alias ya que a mi me gusta alternar entre cat y bat para la lectura de archivos


Fuentes para la Kitty:

Para esto vamos a dirigirnos a descargar las fuentes a la página de Nerd Fonts y descargar las fuentes que más les gusten, en mi caso uso Hack Nerd Font


Una vez descargadas las fuentes procedemos a descomprimir el archivo .zip y copiar las fuentes a la carpeta de fuentes de nuestro sistema:

$ cd  /usr/share/fonts
$ mv  ~/Descargas/HackNerdFont-* .

Finalmente, descomprimimos el archivo y eliminamos el zip

$ unzip HackNerdFont-*.zip
$ rm HackNerdFont-*.zip

Edición de fuente en Kitty

Ahora que ya tenemos las fuentes instaladas procedemos a editar el archivo kitty.conf para cambiar la fuente por la que acabamos de instalar

$ nano ~/.config/kitty/kitty.conf

Dentro del archivo vamos a cambiar la línea de font_family por la fuente que acabamos de instalar

font_family HackNerdFonts

Para mi gusto la letra es muy pequeña, asi que he decidico cambiar el tamaño

font_size 13

Establecer el mapeo de las terminales

En caso de que queramos configurar el mapeo de terminales para que con control y las flechas nos movamos de un lado a otro vamos a hacerlo con la siguiente configuración en la kitty

map ctrl+left neighboring_window left
map ctrl+right neighboring_window right
map ctrl+up neighboring_window up
map ctrl+down neighboring_window down

Ahora con esto cada vez que abramos una nueva terminal con ctrl + shift + enter podemos navegar entre terminales sin usar el mouse para seleccionar la terminal:

Y con supr + shift L podemos cambiar la orientación de las ventanas


Opcional: Clipboard extra

En caso de que queramos tener un clipboard extra para copiar y pegar entre terminales podemos agregar la siguiente configuración en kitty

map f1 copy_to_buffer a # Copiar al buffer 'a' con F1
map f2 paste_from_buffer a # Pegar desde el buffer 'a' con F2

Con esto al hacer ctrl C copias y ctrl V pegas

Pero al usar f1 copias y f2 pegas y asi se manejan diferentes clipboards


Configuración de cursor

Para cambiar la configuración del cursor podemos agregar las siguientes líneas en kitty.conf

cursor_shape beam
cursor_beam_thickness 1.8

Esto es parra que el cursos de la terminal se vea más delgado


Mapeo de terminales con CTRL+SHIFT+Enter o T

Para abrir nuevas terminales con ctrl + shift + enter o ctrl + shift + t podemos agregar las siguientes líneas en kitty.conf

map ctrl+shift+enter new_window_with_cwd
map ctrl+shift+t new_window_with_cwd

Configuración de bindkeys en zsh:

Para mejorar la experiencia en la terminal podemos agregar los siguientes bindkeys en el archivo .zshrc

$ nano ~/.zshrc

Dentro del archivo vamos a agregar las siguientes líneas al final del documento:

#Bindkeys
bindkey "^[[H" beginning-of-line # Inicio de línea
bindkey "^[[F" end-of-line       # Fin de línea 
bindkey "^[[3~" delete-char      # Suprimir
bindkey "^[[1;3C" forward-word   # Ctrl + Flecha derecha
bindkey "^[[1;3D" backward-word  # Ctrl + Flecha izquierda

Con esto ya tendremos los bindkeys configurados para una mejor experiencia en la terminal
