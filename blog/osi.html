Cómo funciona el Modelo OSI

La mayoría de las personas enseñan o aprenden OSI como una lista para memorizar. 7 capas - sus nombres

Esto no es de mucha utilidad ya que esto no te dice por qué existe esa capa o cual problema resuelve.

Entonces, ya entendiendo esto sabemos que las preguntas correctas no son ¿Cuáles son las capas? Sino ¿Qué problema tenía la red que obligó a inventar cada capa?


¿Por qué existen las capas?

Imaginemos que queremos enviar un mensaje a alguien en otra ciudad. Si pone atención podrá observar que hay varios problemas independientes que resolver:


  1. ¿Cómo convierto mi idea en bits? -> Aplicación y presentación
  2. ¿Cómo mantengo la conversación organizada? -> Sesión
  3. ¿Cómo se asegura que el mensaje llegue completo y en orden? -> Transporte
  4. ¿Cómo sé a qué dirección enviarlo si hay miles de caminos posibles? -> Red
  5. ¿Cómo lo mando físicamente al siguiente dispositivo? -> Enlace
  6. ¿Cómo lo convierto en señal eléctrica, luz o radio? -> Física


Como se puede observar, cada capa resuelve un problema distinto e independiente

Esto es importante ya que significa que se puede atacar cada problema por separado, y esto es justo lo que hace un atacante


OSI vs TCP/IP

OSI es el modelo teórico mientras que TCP/IP es lo que existe en la realidad. En la práctica no se hablan de 7 capas sino que se hablan de 4

TCP/IPOSIQué hace?
AplicaciónCapas 5, 6 y 7HTTP, DNS,SSH - Lo que ve el usuario
TransporteCapa 4TCP, UDP - Cómo se mueven los datos
InternetCapa 3IP - Cómo se enrutan los datos | Hacia donde van
Acceso a la redCapas 1 y 2Ethernet, Wi-Fi - Cómo llegan/se envían los datos físicamente

Nota: Wifi como medio de transmisión opera en capas 1 y 2. Lo que a veces se confunde como "Wifi en capa 3" es el protocolo IP corriendo sobre Wifi, pero IP y Wifi son cosas distintas. Esto importa en ofensiva porque un ataque de deautenticación Wifi opera en capa 2, sin necesidad de IP


De igual forma si desean ver las capas de OSI por separado:

7 - AplicaciónHTTP,FTP,IRC,SSH,DNSData
6 - PresentaciónSSL, TLS, MPEG, JPEGData
5 - SesiónAPI, WinSock, SocketsData
4 - TransporteTCP, UDPSegmentos
3 - RedIP, ICMP, IPSec, IGMPPaquetes
2 - EnlaceEthernet, PPP, ATMFrames
1 - FísicaCoax, Ethernet, Wireless, USBBits

Cuando se está hackeando, la mayoría del tiempo operas en capa 3 y 4. Cuando haces ARP spoofing, operas en capa 2. Cuando explotas una aplicación web, operas en capa 7 pero el ataque viaja por todas las demás


Qué pasa cuando escribes google.com?

Estos son ejercicios mentales que ayudan o le obligan a entender todas las capas en un contexto real:


  1. Su navegador es capa 7. Genera un HTTP request: GET / HTTP/1.1
  2. Necesita saber la IP de google.com, asi que hace una consulta DNS, que también es capa 7 pero usa UDP en capa 4
  3. Una vez ya tiene la IP, en capa 4 inicia el handshake con esa IP para establecer la conexión antes de mandar cualquier dato
  4. La IP en capa 3 decide por qué camino mandar cada paquete. Esta capa no sabe nada de TCP ni de HTTP, solo sabe o se enfoca en direcciones
  5. Ethernet en capa 2 toma ese paquete y lo manda al siguiente dispositivo físico, que normalmente es un router. Al ser capa 2 hay que recordar que se hace uso de direcciones MAC no de IPs
  6. Su router recibe la trama en capa 2, sube a capa 3 para leer la IP de destino, decide donde mandarlo y vuelve a bajar a capa 2 para mandarlo al siguiente salto

Ese proceso de subir y bajar capas se repite en cada dispositivo(nodo) del camino hasta llegar a Google


Cómo se ve esto en la ofensiva?

Con todo lo anterior visto podemos entender que cada capa es una superficie de ataque independiente:

  • Capa 7: Cualquier vulnerabilidad de aplicación - SQL injections o XSS
  • Capa 4: Abusa el hecho de como TCP maneja las conexiones - SYN flood, port scanning, session hijacking
  • Capa 3: Manipula hacia dónde van los paquetes
  • Capa 2: Uno se coloca entre 2 dispositivos - MAC flooding

Objetivo Práctico

Cuando uno está aprendiendo este tipo de cosas lo mejor que se puede hacer es llevar la teoría a la práctica para entender como realmente funcionan las cosas

En wireshark nos vamos a dedicar a crear una captura de red a un sitio HTTP para analizarla de una manera más detenida. Dentro de esta captura vamos a buscar:

  1. Una trama de Ethernet: Consiste en buscar las direcciones MAC tanto de origen como destino
  2. Un paquete IP: Buscar ahora la IP de origen y destino
  3. Un segmento TCP: Buscar información como puerto origen, destino y sus flags
  4. El HTTP request - texto plano del GET
